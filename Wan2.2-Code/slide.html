0 · 全景

一句话理解整个系统

1
编码

用 UMT5 把 prompt 编码成语义向量；图生视频时再用 VAE 把首帧编码成 latent。

2
去噪

从纯高斯噪声出发，DiT (WanModel) 在压缩后的 latent 空间迭代去噪 50 步，文本作为 cross-attention 条件。

3
解码

用 VAE Decoder 把去噪完成的 latent 还原成像素视频，存成 mp4。

Wan2.2 = Latent Diffusion + DiT + Flow Matching。本文选最干净的 TI2V-5B（稠密单模型）作主线，一个模型同时走通「文生视频 t2v」与「图生视频 i2v」两条路径。
0 · 全景

系统总览

文本(UMT5)与可选首帧(VAE)编码为条件 → DiT 在 latent 空间迭代去噪 50 步(每步含 CFG 双前向) → VAE 解码出片

1.3 · 核心

TI2V-5B 生成全流程

W=1280 · H=704F=121vae_stride (4,16,16) patch (1,2,2)z_dim 48dim 3072
重点看 DiT 内部三路接线：噪声 latent 经 patch_embedding → Self-Attn(Q/K/V)；文本 context → Cross-Attn(K/V)；时间步 → AdaLN 调制注入各 Block。调度器只接收 noise_pred。
1.3 · 核心

逐张量形状追踪

① 文本编码 UMT5-XXL：prompt 与负向 prompt 各编码一次 → context: [L≤512, 4096]
▼
② 算 latent 形状 像素按 vae_stride 压缩 → (z_dim, (F-1)//4+1, H//16, W//16) = (48, 31, 44, 80)
▼
③ 算 DiT 序列长度 latent 再按 patch(1,2,2) 切块展平 → seq_len = 31 × (22×40) = 27,280 token
▼
④ 初始化噪声 noise ~ N(0,1) 形状 (48, 31, 44, 80) —— 要去噪的「视频 latent」
▼
⑤ 去噪循环 ×50 每步 DiT 跑两次(cond/uncond) 做 CFG → scheduler.step() 把 latent 往 t→0 推进
▼
⑥⑦ 解码存盘 VAE 解码 (48,31,44,80) → (3,121,704,1280) → save_video 写 mp4 @24fps
1.3 · 核心

去噪主循环 与 CFG 精髓

context      = self.text_encoder([input_prompt], self.device)   # 条件
context_null = self.text_encoder([n_prompt],     self.device)   # 无条件/负向

latents = noise
for t in tqdm(timesteps):                  # ⑤ 50 步去噪
    noise_pred_cond   = self.model([latents[0]], t=t, context=context)[0]
    noise_pred_uncond = self.model([latents[0]], t=t, context=context_null)[0]
    # ★ Classifier-Free Guidance：放大「条件 − 无条件」的方向
    noise_pred = noise_pred_uncond + guide_scale * (noise_pred_cond - noise_pred_uncond)

    latents = [sample_scheduler.step(noise_pred, t, latents[0])[0]]

videos = self.vae.decode(latents)        # ⑥ 解码回像素
代价：每步两次 DiT 前向。 s=guide_scale=5.0 越大越贴合 prompt 但可能过饱和。50 步实际是 100 次 DiT 前向 —— 这是推理耗时的主要来源。
2 · 文本编码

文本编码器 UMT5-XXL

umt5-xxl encoder-only层数 24dim 4096heads 64权重 11GB bf16
def __call__(self, texts, device):
    ids, mask = self.tokenizer(texts, return_mask=True, add_special_tokens=True)
    seq_lens = mask.gt(0).sum(dim=1).long()        # 每条文本真实长度
    context = self.model(ids, mask)                  # [B, 512, 4096]
    return [u[:v] for u, v in zip(context, seq_lens)]  # 去 padding → [L, 4096]
UMT5 用 相对位置编码 + T5 风格无缩放注意力 + RMSNorm，Wan 直接复用。--t5_cpu 让这个 11GB 编码器常驻 CPU 省显存；编码完成后立即 .cpu() 把显存腾给 DiT。
3 · DiT 主干

WanModel 数据流

带噪 latent 经 patch_embedding 作 Self-Attn 的 Q/K/V；文本 context 经 text_embedding 只作 Cross-Attn 的 K/V；时间步 t 经 MLP 生成 6 路 AdaLN 调制，仅作用于 Self-Attn 与 FFN 的归一化（不进 Cross-Attn）。Head+unpatchify 还原成与输入同形的速度场。
3.1 · DiT

三类 Embedding

# A. patch_embedding：3D 卷积把 latent 切块，等价 ViT patchify
self.patch_embedding = nn.Conv3d(in_dim, dim, kernel_size=(1,2,2), stride=(1,2,2))
        # in_dim=48 → dim=3072 ;  (48,31,44,80) → (3072,31,22,40) → 27280 token

# B. text_embedding：把 4096 维文本投到 3072
self.text_embedding  = nn.Sequential(nn.Linear(4096, dim), nn.GELU(), nn.Linear(dim, dim))

# C. time_embedding：正弦 → MLP ；time_projection 再投到 6 份调制参数
self.time_projection = nn.Sequential(nn.SiLU(), nn.Linear(dim, dim * 6))  # ★ ×6
「逐 token 时间步」是 TI2V 的关键设计。 普通 DiT 一个样本一个标量 t；这里 t 形状是 [B, seq_len] —— 每个 token 可有不同时间步，让「首帧已知(t=0)、其余待生成(t=T)」能在同一次前向里混合，正是图生视频的底层机制。
3.2 · DiT

AttentionBlock 与 AdaLN 调制

e = (self.modulation + e).chunk(6, dim=2)  # shift/scale/gate ×2

# ── 自注意力 (时空全局 + RoPE) ──
y = self.self_attn(self.norm1(x)*(1+e[1])+e[0])
x = x + y * e[2]                  # gate

# ── 交叉注意力 (文本) + FFN ──
x = x + self.cross_attn(self.norm3(x), context)
y = self.ffn(self.norm2(x)*(1+e[4])+e[3])
x = x + y * e[5]                  # gate
全局 e0 进 Block 后 + modulation 再 chunk(6) 得本层专属 e[0..5]；分别控制 Self-Attn 与 FFN 的归一化/门控；Cross-Attn 不吃调制
AdaLN 调制把「扩散时间步 t」注入每层：norm(x)·(1+scale)+shift 的 FiLM 仿射 + gate 门控残差，让同一套权重在不同噪声水平下表现不同去噪行为。注意 Wan2.2 不是 AdaLN-Zero —— 它只把输出层 head.head 置零来稳训练。
3.3 · DiT

注意力实现 + 3D RoPE

注意力底层 flash_attention 优先级 FA3 → FA2 → PyTorch SDPA，用 varlen 接口处理变长。

# 把 head_dim 切成 3 段，分别编码 帧/高/宽 三个轴的位置
freqs = freqs.split([c-2*(c//3), c//3, c//3], dim=1)   # 时间/H/W
freqs_i = torch.cat([
    freqs[0][:f]...expand(f,h,w,-1),   # 帧轴
    freqs[1][:h]...expand(f,h,w,-1),   # 高轴
    freqs[2][:w]...expand(f,h,w,-1)],  # 宽轴 dim=-1)
x_i = torch.view_as_real(x_i * freqs_i).flatten(2)       # 复数乘 = 旋转
3D RoPE 让模型天然理解 token 在「第几帧、第几行、第几列」的三维位置，对分辨率/帧数有外推能力。只作用于自注意力的 q/k；交叉注意力不需要（文本无空间位置）。
3.4 · DiT

Head 与 unpatchify：DiT 是个沙漏

patch_embedding 下采样+升维，Head+unpatchify 是它的精确逆操作 —— 一压一还原，形状回到原点
关键常被忽略：Head 的输出维度不是 48，而是 48×prod(patch)=48×(1·2·2)=192 —— 每个 token 吐出「它当初代表的那个 1×2×2 小块里全部 48 通道的数值」，unpatchify 再靠 grid_sizes=(31,22,40) 把这 192 个数 view+einsum+reshape 拆回小块、贴回空间位置，于是 22×40 的「块网格」展开回 44×80 的「像素网格」。
4 · VAE

高压缩 VAE 2.2

类 Wan2_2_VAEz_dim 48压缩 时4×空16×16额外 patchify=2权重 2.7GB
patchify(2×2) + Encoder(空间8×下采样) 把像素压成 48 维 latent；Decoder 对称还原。全程因果卷积 + 分块 + feat_cache，合计空间 16×16、时间 4× 压缩
压缩比怎么来的？ VAE 网络本身做 8×8 空间下采样，encode 入口先 patchify(patch=2) 做 2×2 像素打包 → 合计 16×16 空间压缩；时间维由 temporal 下采样实现 4× 压缩。这就是 vae_stride=(4,16,16) 的物理含义，也是 5B 模型能生成 720p/5 秒视频的关键。
4.1 · VAE

因果卷积 与 分块编解码

CausalConv3d（只向过去 padding）
class CausalConv3d(nn.Conv3d):
  self._padding = (W,W,H,H, 2*pad_t, 0)
       # ★ 时间维只往「过去」pad，右侧=0

  def forward(self, x, cache_x=None):
    if cache_x is not None:    # 拼上一块尾帧
      x = torch.cat([cache_x, x], dim=2)
    return super().forward(F.pad(x, padding))
encode（分块循环）
x = patchify(x, patch_size=2)        # 2×2→12ch
iter_ = 1 + (t-1)//4             # 按 4 帧分块
for i in range(iter_):
    out_i = self.encoder(块i, feat_cache=...)
mu = (mu - scale[0]) * scale[1]      # ★ 归一化
return mu                          # 推理只取均值
因果性 + feat_cache：视频 VAE 保证「当前帧只依赖过去帧」，块间用 feat_cache 传递边界帧卷积中间结果保证拼接无缝。scale=[mean,1/std] 把 latent 标准化到利于扩散的尺度，这是 LDM 的标准做法。
5 · 调度器

流匹配调度器 UniPC

step() 每步：convert → (非首步)UniC 校正 → 入缓存 → 定阶数 → UniP 预测 → step+1，整段无可学习参数，纯数值积分
Flow Matching 直觉：不预测「加了多少噪声」，而学习从噪声到数据的速度场。σ 从 1(纯噪声) 线性降到 0(干净)，timestep = σ×1000。
sigmas = np.linspace(1, 0, num_steps)
# shift 把采样重心往高噪声端偏移
sigmas = shift*sigmas/(1+(shift-1)*sigmas)
                              # shift=5.0
UniPC = 多步预测-校正求解器，用历史几步做高阶外推，50 步即达朴素 Euler 上百步质量。
6 · CFG

Classifier-Free Guidance

每个去噪步把 DiT 前向跑两次(条件/无条件)，再沿两者之差外推 s 倍 —— 这是 prompt 贴合度的来源，也是推理耗时翻倍的原因
v_guided = v(x_t,t,∅) + s·[ v(x_t,t,c) − v(x_t,t,∅) ]
s=1 退化为普通条件采样；s>1 放大 prompt 影响(默认 5.0)；「无条件」用的是负向 prompt，故 CFG 同时「靠近正向、远离负向」。

以扩散时间步 boundary(t2v=875) 为界：高噪声步用 high-noise 专家(搭大结构)、低噪声步用 low-noise 专家(雕细节)，CFG 强度随之切换；offload 模式下显存只常驻当前专家
「MoE」的确切含义：不是层内 token 级 FFN 路由，而是整模型级两个专家按扩散阶段分工 —— 每步只激活一个，故「总参 27B、单步激活 14B」。主干仍复用同一个 WanModel。

11 · 术语

术语速查 & 小结

术语含义
LatentVAE 压缩后的低维表示；1280×704×121 像素 → 80×44×31×48，扩散在此小空间进行。
DiTDiffusion Transformer，用 Transformer 替代 U-Net 做去噪主干 (即 WanModel)。
Flow Matching学习从噪声到数据的速度场，σ 从 1→0，比 DDPM 采样更高效。
CFG用「条件 vs 无条件」预测之差放大 prompt 影响，每步两次前向。
AdaLN时间步经调制向量 (shift/scale/gate) 控制每层归一化；Wan2.2 只把 head.head 置零稳训练。
3D RoPE按帧/高/宽三轴分别编码的旋转位置编码。
MoE 双专家A14B 的 high/low-noise 两个 DiT 按 boundary 切换；TI2V-5B 不用。
FSDP / Ulysses分别切「模型参数」/「序列长度」的两种多卡并行，可叠加。
读懂 TI2V-5B 这一条数据流，其余任务只是「加料」。 🎬
